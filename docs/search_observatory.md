# MindReply Search Demand Observatory

This is the internal SEO and demand-positioning contract for MindReply. It guides future agents and reports without making public claims that are not backed by evidence.

## Demand Position

MindReply should compete where the buyer has urgent pain and few focused providers:

- Website buying-friction rescue: the page exists, but the offer does not close.
- Response overload: messages, follow-ups, and decisions are taking too long.
- Client-facing pressure: the next reply must be calm, clear, and commercially useful.
- Assisted close: the buyer needs a human/invoice route without a bloated sales process.
- Privacy-safe proof: sensitive work needs receipts, not public exposure.

## Public Vocabulary

Use premium, plain language:

- "Website Completion Package"
- "ranked action queue"
- "send-ready copy"
- "assisted close"
- "privacy-safe receipt"
- "one clear next move"
- "buying-friction rescue"
- "response overload"

Avoid public claims about staff count, active agents, revenue, subscriptions, security access, or deployment state unless verified by files, workflow runs, connectors, or live endpoints.

## Locale And Country Handling

The site should use `/api/geo-locale` to infer the visitor country from Vercel or edge headers and recommend one of ten supported language options:

- English
- Spanish
- French
- German
- Italian
- Portuguese
- Arabic
- Japanese
- Korean
- Chinese

The selector must stay subtle. It should help international buyers without interrupting the main conversion path.

## Observatory Loop

Each report or implementation pass should check:

1. Top message still names urgent pain, not abstract technology.
2. Primary CTA points to MRagent or the Website Completion Package.
3. Secondary CTA exposes full package details or invoice/payment route.
4. Metadata includes buying friction, response overload, assisted close, and privacy-safe receipt language.
5. Footer includes trust, demand lanes, and contact path without clutter.
6. Mobile layout has no text/button overflow and no invisible next step.
7. Language support remains country-aware and manually changeable.
8. No public page invents revenue, live integrations, or agency/staff scale.

## Automation Boundary

Search observation may recommend changes. It must not auto-publish claims based on unverified keyword guesses. Any public copy change should be small, visible, and tied to conversion clarity or owner-approved demand positioning.
