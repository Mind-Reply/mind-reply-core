# MindReply Owner Decision Security

This file defines the security and owner-decision boundary for MindReply work.

## Private Owner Channel

Sensitive audit data, deployment evidence, security findings, and operational blockers may be summarized for the configured owner report address. They must not be copied into public website copy.

Public website contact remains:

- `info@mind-reply.com`

Private owner reporting may use configured secrets or connector-approved destinations. Do not expose private owner inboxes in public pages, metadata, screenshots, reports, or marketing copy.

## Defensive Security Boundary

Security work is defensive and owner-controlled.

Allowed:

- secret exposure checks
- deployment and domain verification
- auth, payment, webhook, and receipt safety checks
- privacy redaction checks
- evidence-backed owner decision reports
- risk labels and recommended owner actions

Not allowed:

- public claims of hidden security teams or fake active staff
- publishing raw private pressure text
- exposing API keys, tokens, private addresses, or personal inboxes
- claiming revenue, billing, subscriptions, Pro status, or integration access without connector, payment, invoice, or owner-confirmed proof

## Owner Decision Format

Every owner decision report should stay concise:

- status: green, amber, or red
- what changed
- top blocker
- next revenue move
- owner decision needed
- Website Completion Package status
- live deployment proof or local-only proof label
- email and Slack delivery receipt when available

## Public Copy Rule

Public copy should be factual, warm, premium, and easy to understand. It should never use theatrical agent language, internal role names, private owner details, or unverifiable claims.
